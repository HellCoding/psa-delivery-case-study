# WBS 예시

| 단계 | 기간 예시 | 작업 | 담당 | 산출물 |
|---|---|---|---|---|
| Discovery | Week 0 | PSA 업무 범위, 사용자 역할, 제외 범위 확인 | PM/PO | Scope 요약 |
| PRD | Week 0-1 | FR/BR, 화면 목록, 데이터 모델, acceptance criteria 작성 | PM/PO | 공개용 PRD |
| Planning | Week 1 | Phase Epic, Feature Task, QA scenario 초안 구성 | PM | Jira backlog |
| Foundation | Week 1 | ODC 앱/모듈, role, navigation, 공통 레이아웃 준비 | Vendor | Foundation build |
| Core Data | Week 1-2 | BizOpty, Project, Task, Employee, ProjectMember 구현 | Vendor | Core CRUD |
| Timesheet | Week 2-3 | Daily Entry, Weekly Grid, Submit Workflow 구현 | Vendor | Timesheet flow |
| Approval | Week 3 | Approve/Reject, 상태 동기화, RejectReason 구현 | Vendor | Approval flow |
| Dashboard | Week 3-4 | PM/Executive dashboard, KPI, period filter 구현 | Vendor | Dashboard |
| Report Builder | Week 4 | Report definition, field/filter/grouping, execution 구현 | Vendor | Report module |
| QA/UAT | Week 4-5 | Priority 1 시나리오 실행, 결함 등록, 재검증 | PM/QA/Vendor | QA report |
| Quality Gate | Week 5 | ODC Mentor findings 검토와 수용 기준 정리 | PM/Reviewer | Code quality report |
| Decision | Week 5 | 벤더 비교, baseline 결정, 후속 backlog 작성 | PM/Stakeholder | Decision log |
