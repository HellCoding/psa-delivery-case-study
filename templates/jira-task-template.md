# Jira Task 템플릿

Task는 실제 작업자가 바로 수행할 수 있을 정도로 구체적이어야 합니다. “개선하기”처럼 추상적인 표현보다, 입력값, 처리 조건, 완료 기준을 명확히 작성합니다.

## 요약

작업 내용을 한 문장으로 작성합니다.

## 배경

이 작업이 필요한 이유를 작성합니다.

- 관련 요구사항:
- 관련 Story/Epic:
- 현재 문제:

## 작업 내용

- 변경할 화면/기능:
- 검증해야 할 조건:
- 예외 처리:

## 인수 기준

- [ ] Given / When / Then 형식으로 작성
- [ ] 정상 케이스가 검증된다.
- [ ] 예외 케이스가 검증된다.
- [ ] 관련 QA 시나리오와 연결된다.

## QA 시나리오 연결

- 시나리오 ID:
- 테스트 담당:
- 예상 결과:

## 의존성

- 선행 작업:
- 정책 결정:
- 외부 확인 필요 사항:

## 완료 체크리스트

- [ ] 작업 완료
- [ ] 리뷰 완료
- [ ] QA/UAT 통과 또는 결함 등록
- [ ] 관련 문서 갱신
- [ ] 후속 backlog 필요 여부 확인
